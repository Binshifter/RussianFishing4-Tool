</div>

![img.png](img.png)

## 环境要求

- **操作系统**：Windows 10/11
- **Python 版本**：Python 3.9+（GUI 运行推荐 Python 3.12，OCR 依赖需安装在 Python 3.12）
- **游戏要求**：Russian Fishing 4 窗口化运行，游戏内鼠标灵敏度调至最低

## 依赖安装

```bash
# 基础依赖
pip install -r requirements.txt

# OCR 文字识别依赖（鱼护计数、鱼名、重量识别）
pip install rapidocr_onnxruntime opencv-python-headless
```

| mss | 10.2.0 | 屏幕截图 |

## 功能说明

### 原版功能

- **自动钓鱼**：模板匹配检测鱼图标、ready 文字、鱼上钩状态，自动完成抛竿→收线→入户/放生循环
- **开启自动进食**：脚本每隔约一段时间自动按一次游戏内的 4 键，需要把食物放在 4 键上
- **达标重量倍数**：AI 判断（对接火山引擎）鱼获的筛选标准，填 1 表示只保留达到标准重量的鱼，0 表示强制保留所有鱼入户
- **收线模式**：平收、JIG 模式、海钓模式
- **开启自动装配引线/拟饵**：防止引线被咬断，引线公斤数和拟饵型号从模板文件夹自动扫描
- **开启重登回城卖鱼**：小退后执行固定脚本往返卖鱼点和钓鱼点
- **大鱼超时自动重登卖鱼**：连续收线超时达到设定次数后自动重登卖鱼
- **时间策略配置**：每日运行时长、连续运行时长、小休息范围，用于防检测

### 相比原版新增功能

#### 1. ONNX OCR 文字识别
- 集成 PP-OCRv4 模型，使用 rapidocr_onnxruntime 进行文字识别
- 支持鱼名、重量（克/千克/公斤/kg/g）自动识别，替代部分模板匹配
- 鱼护计数 OCR：自动识别屏幕右下区域的 `X/Y` 格式数字（如 78/150），识别后自动同步到初始渔户数量输入框
- 识别区域限定为屏幕中心横竖交叉的右下四分之一区域，减少干扰

#### 2. 34 种钓鱼预设配置
- 路亚 (Spin) 系列：Spin 默认、带暂停、带提竿
- Beluga Venga 10000 系列：遛狗步法、抽动钓法、跳底钓法
- 底钓 (Bottom) 系列：c6/c12/c17/c30/c35 多种配置
- 海钓 (Pirk) 系列：强力抽竿、移动、抽竿等多种深度配置
- 浮钓 (Float) 系列
- 选择预设后自动切换收线模式并填充对应参数（卷线时长、停歇时长、沉底时间、抛竿延迟等）

#### 3. 遛狗模式 (Troll)
- 新增狗收线模式，支持抽动间隔和抽动时长参数配置
- 与 JIG 模式独立，参数互不干扰

#### 4. 模式专属参数验证
- 普通模式：仅检查落水时间 > 0
- JIG/遛狗模式：要求卷线时长和停歇时长 > 0
- 海钓模式：要求海钓参数（沉底时间、抛竿延迟等）> 0
- 各模式参数独立校验，避免跨模式误报

#### 5. 入户记录面板
- 右侧运行控制区新增"入户记录"面板
- 实时记录每条入户鱼的完整信息，包含时间戳
- 格式：`[21:10:39] 【入户】中鱼强制入户 | 欧鳊 | 1.737kg | 渔户94/150`

#### 6. 快捷键控制
- `Ctrl+P`：暂停钓鱼
- `Ctrl+R`：继续钓鱼
- `Ctrl+Q`：结束钓鱼
- 修复暂停功能因键盘重复触发导致自动恢复的问题

#### 7. 大休息（睡眠）策略
- 支持大休息时间配置，到期后自动下线
- 大休息结束后自动检测登录按钮并重新上线
- 上线后自动恢复卖鱼流程并回到钓点

#### 8. 人类行为模拟
- 随机鼠标微动、随机按键，模拟真人挂机行为
- 降低被检测风险

## 使用说明

1. 脚本启动前需要先配置好路亚，包括摩擦和转速
2. 游戏窗口化运行，并且将游戏内控制中的鼠标灵敏度调整至最低
3. 鼠标灵敏度调整至最低后，如果重登脚本对不上，可能需要去 map_config.json 中调整对应点位的 initial_rotation 与 water_rotation 的值
4. 不要在程序运行过程中（特别是重登操作时）去动鼠标，建议挂机时拔掉鼠标
5. OCR 功能需要安装 rapidocr_onnxruntime，首次启动会下载 ONNX 模型文件
